# 3. Architectural Design Laws

## Typography Governance

Forbidden:
✗ Inter
✗ Roboto
✗ Open Sans
✗ Default system fonts

Required:

• 1 High-impact display typeface (technical/editorial)
• 1 Controlled body font
• 1 Numeric-focused font (if data-heavy)

Typography must:

• Establish authority via scale contrast
• Create vertical rhythm tension
• Avoid uniform sizing
• Use letter-spacing strategically

---

## Color Sovereignty System

All colors via CSS variables only.

Hierarchy:

• Deep Dominant Base (not flat black)
• Structured Surface Layers (3 tonal steps minimum)
• One Strategic Accent (electric, metallic, or deep signal tone)
• Muted Functional Secondary
• Controlled grayscale ladder

Avoid:

✗ Evenly balanced palettes  
✗ Startup purple gradients  
✗ Pastel SaaS aesthetics  

Dark UI must feel layered, not flat.

---

## Layout Intelligence

No centered SaaS symmetry.
No predictable 3-card grids.

Use:

• Intentional asymmetry
• Hierarchical grouping logic
• Variable spacing tension
• Edge-aligned composition
• Sectional contrast shifts

Whitespace must create controlled pressure.

---

## Surface Engineering

Allowed:

• Gradient strokes (subtle, directional)
• Inner shadows for panel depth
• Accent edge lighting
• Subtle grain texture overlays
• Layered transparency (controlled)

Not allowed:

✗ Random blur spam  
✗ Glassmorphism clichés  
✗ Default shadow presets  

Depth must communicate structure.

---

## Motion Command System

Motion exists to communicate:

• State change
• Data refresh
• Focus
• Panel priority

Permitted:

• Sequenced panel reveal
• Directional hover activation
• Accent sweep highlights
• Controlled elevation shifts

Forbidden:

✗ Decorative bouncing  
✗ Infinite looping distractions  
✗ Excessive micro-animations  

Serious product → serious motion discipline.
